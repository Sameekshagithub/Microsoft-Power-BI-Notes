01
Introduction to Power BI

Power BI is a Business Intelligence (BI) and Data Visualization tool developed by Microsoft. It enables users to connect to various data sources, transform raw data, build data models, and create interactive reports and dashboards.

🖥️ Power BI Desktop

Windows application for building reports, models, and DAX calculations locally.

☁️ Power BI Service

Cloud-based (app.powerbi.com) for publishing, sharing, and collaborating on reports.

📱 Power BI Mobile

iOS/Android app for consuming reports and dashboards on the go.

🔗 Power BI Gateway

On-premises data gateway for scheduled refreshes of local data in the cloud.

📦 Power BI Embedded

Embed Power BI reports in custom applications using REST APIs.

📋 Power BI Report Server

On-premises report server for organizations not using cloud.

💡 Key Benefit
Power BI allows non-technical users to create rich, interactive analytics without writing complex code — turning raw data into actionable insights.

📌 Import vs DirectQuery vs Live Connection
  • Import Mode: Data is loaded into Power BI's in-memory engine. Fastest performance, scheduled refresh needed.
  • DirectQuery: Queries run directly on the source each time. Always current, but slower.
  • Live Connection: Used for SSAS or Power BI datasets — no local model created.
04
Power Query & Data Transformation (M Language)

Power Query Editor is where you clean, reshape, and transform data before loading it into the model. Each step is recorded and generates M code behind the scenes.

  • Remove Rows/Columns: Drop nulls, errors, duplicates
  • Change Data Types: Text, Number, Date, Boolean, etc.
  • Split/Merge Columns: By delimiter, by position
  • Filter Rows: Condition-based filtering
  • Group By: Aggregate data (SUM, COUNT, AVG)
  • Pivot / Unpivot: Reshape table structure
  • Merge Queries: JOIN-like operation between tables
  • Append Queries: UNION-like stacking of tables
  • Add Custom Column: Write M expressions
  • Conditional Column: if-then-else logic without code
M Language Example
// Custom Column: Full Name
= Table.AddColumn(Source, "Full Name",
  each [First Name] & " " & [Last Name],
  type text)

// Filter rows where Sales > 1000
= Table.SelectRows(Source,
  each [Sales] > 1000)
05
Data Modeling

The data model defines how tables relate to each other. A well-structured model is essential for accurate DAX calculations and report performance.

⭐ Star Schema

Fact table at center, dimension tables around it. Best practice for Power BI models.

❄️ Snowflake Schema

Normalized dimensions. More complex but reduces redundancy. Works but can slow DAX.

ConceptDescription
RelationshipLinks two tables via a common key (1:Many, Many:1, 1:1, Many:Many)
CardinalityOne-to-Many (most common), Many-to-Many, One-to-One
Cross Filter DirectionSingle (default) or Both — controls how filters propagate
Fact TableContains measurable events/transactions (Sales, Orders, etc.)
Dimension TableDescriptive context (Date, Customer, Product, Region)
Calculated ColumnComputed row by row at refresh time, stored in model
MeasureComputed dynamically in DAX based on filter context
HierarchyDrill-down paths (Year → Quarter → Month → Day)
✅ Best Practices
Always use a Date Table. Mark it as Date Table. Avoid Many-to-Many unless necessary. Use surrogate keys. Hide foreign key columns.
06
DAX – Data Analysis Expressions

DAX is a formula language used to create calculated columns, measures, and tables. It is context-sensitive — the same formula can return different results based on filters applied.

🔑 Two Key Contexts
Row Context: Evaluates formula row by row (calculated columns, iterators like SUMX).
Filter Context: Set of filters active at the time of evaluation (slicers, row/column headers, CALCULATE).
Basic Measures
-- Total Sales
Total Sales = SUM(Sales[SalesAmount])

-- Average Order Value
Avg Order = AVERAGEX(Orders, Orders[Amount])

-- Count of Customers
Customer Count = DISTINCTCOUNT(Sales[CustomerID])
CALCULATE — Most Important DAX Function
-- Sales for specific category
Laptop Sales = CALCULATE(
  [Total Sales],
  Products[Category] = "Laptop"
)

-- Remove all filters
All Sales = CALCULATE([Total Sales], ALL(Sales))
Time Intelligence
-- Year To Date
YTD Sales = TOTALYTD([Total Sales], 'Date'[Date])

-- Previous Year
PY Sales = CALCULATE([Total Sales],
  SAMEPERIODLASTYEAR('Date'[Date]))

-- Month over Month Growth
MoM % = DIVIDE([Total Sales] - [PM Sales], [PM Sales])

07
Visualizations

Power BI offers a wide range of built-in visuals plus custom visuals from the AppSource marketplace.

📊 Bar / Column Chart

Compare categories. Clustered, stacked, 100% stacked variants.

📈 Line / Area Chart

Show trends over time. Area shows cumulative values.

🥧 Pie / Donut Chart

Show proportions of a whole. Best for ≤ 5 categories.

🗺️ Map / Filled Map

Geographic distribution. Uses Bing maps by default.

📋 Table / Matrix

Tabular data with drill-down. Matrix supports row/column pivoting.

🃏 Card / Multi-row Card

Display a single KPI value prominently on the report.

💡 KPI Visual

Shows metric vs target with trend indicator.

🌲 Treemap

Hierarchical data as nested rectangles sized by value.

🔵 Scatter / Bubble

Show correlation between two measures. Bubble adds a third dimension.

🍩 Gauge

Circular gauge showing progress toward a target.

🎚️ Slicer

Interactive filter control — dropdown, list, slider, date range.

🌊 Waterfall Chart

Show incremental changes (positive/negative) to reach a final value.

🎨 Formatting Tips
Use Conditional Formatting on tables/matrices. Set Data Labels strategically. Use Themes (View → Themes) for consistent branding. Enable Tooltips for extra context on hover.

Power BI Service (Cloud)

After building in Desktop, you publish to Power BI Service (app.powerbi.com) for sharing and collaboration.

📂 Workspaces

Collaborative spaces (My Workspace + shared workspaces). Teams can share reports and datasets.

📊 Dashboards

Collection of pinned tiles from multiple reports. Single page, always refreshed view.

🔄 Scheduled Refresh

Auto-refresh datasets on schedule (up to 8x/day on Pro, 48x/day on Premium).

📧 Subscriptions

Send automatic email snapshots of reports/dashboards to users on a schedule.

🏢 Apps

Package multiple reports into a polished app to distribute to a broader audience.

🔔 Alerts

Set data-driven alerts on card/gauge tiles — get notified when threshold is crossed.

Security – Row Level Security (RLS)

RLS restricts data access for specific users. It is defined in the data model using DAX filters on tables.

1
In Desktop: Modeling → Manage Roles → Create a role with a DAX filter.
2
Example DAX filter on Sales table: [Region] = USERNAME()
3
Test with Modeling → View As Role to verify data is filtered correctly.
4
Publish to Service → Dataset → Security → Assign users/groups to roles.
🔐 Dynamic RLS
Use USERPRINCIPALNAME() or USERNAME() to dynamically filter data based on the logged-in user's email, using a mapping table.

Pro Tips & Quick Reference
⚡ Performance Tips
  • Use Import mode over DirectQuery
  • Avoid calculated columns — prefer measures
  • Reduce cardinality on columns
  • Limit visuals per page (max 8–10)
  • Disable Auto Date/Time
🎯 DAX Best Practices
  • Always use DIVIDE() instead of /
  • Use variables (VAR) to simplify formulas
  • Prefer measures over calculated columns
  • Use SELECTEDVALUE for single-selection
  • Format measures with units
